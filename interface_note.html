인터페이스 요약 노트

1. 인터페이스 기본 의미

• 규칙(메서드 형태)만 정의, 구현은 하지 않음
• 구현 클래스는 반드시 public으로 구현해야 함
• 오버라이딩 시 접근 제어자가 좁아지면 오류

2. 인터페이스의 역할

• 여러 클래스의 규칙 통일
• 다형성 제공 (상위 타입 역할)
• 다중 상속 가능 → 클래스 상속의 한계 보완

3. 인터페이스 → 추상 클래스 → 구현 클래스 구조

[ Interface ] : 규칙 제공
↓
[ Abstract Class ] : 공통 코드 묶기 (선택)
↓
[ Class ] : 실제 기능 구현
• 추상 클래스는 필요할 때만 사용
• 인터페이스는 항상 가장 상단의 역할을 담당

4. default / static 메서드

• default: 인터페이스에 기본 구현 제공 (예외적 기능)
• static: 인터페이스 이름으로 호출하는 정적 메서드

5. 핵심 요약 (암기판)

■ 인터페이스 = 규칙만
■ 추상 클래스 = 공통 코드 + 규칙 일부
■ 구현 클래스 = 실제 동작

인터페이스 → 공통 규칙
추상 클래스 → 공통 구현
클래스 → 실제 기능
